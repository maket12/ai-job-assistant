## 📝 ТЗ: AI Job Assistant "Kiko"

### 1. Общее описание
Бот для автоматизации поиска работы. Система парсит вакансии, сопоставляет их с профилем пользователя (Резюме + Настройки), рассчитывает точность совпадения (**Match Score**) и генерирует краткое резюме вакансии (**AI Summary**) вместе с персонализированным сопроводительным письмом.

---

### 2. Функциональные модули

#### 2.1. Модуль пользователя (User Settings)
* Хранение данных пользователя: ID, username, язык.
* Хранение файла резюме (PDF/Docx) и его текстового представления.
* **Конфиг поиска:** Навыки (список), Грейд (Junior/Middle+), Локация, Тип занятости.

#### 2.2. Модуль Вакансий (Vacancies & Parser)
* **Парсер:** Периодический сбор данных с HH.ru (в будущем — другие источники).
* **Общая таблица вакансий:**
    * `id`, `title`, `company`, `source` (HH.ru), `salary`, `location`, `description`, `url`, `created_at`.

#### 2.3. AI-Анализатор (Matching Engine)
Центральная часть системы. Для каждой новой вакансии и каждого активного пользователя рассчитывается:
* **Match Score (0-100%):**
    * *Весовые коэффициенты:* Например, совпадение по ключевому стеку (Python) — 50%, по грейду — 20%, по локации — 15%, по остальным навыкам — 15%.
* **AI Summary:** Короткий текст (3-4 предложения), почему вакансия подходит (например: "У вас отличный опыт в FastAPI, что является ключевым требованием здесь").
* **Cover Letter:** Готовый текст письма, адаптированный под требования вакансии и опыт пользователя.

### 3. Интерфейс Бота (UI/UX)

#### Меню "Vacancies" (Листалка):
Реализуется через Inline-кнопки (Влево/Вправо).
* **Экран вакансии содержит:**
    * Заголовок: `Название` в `Компания`.
    * Условия: `Зарплата` | `Локация`.
    * **Match Score:** <code>[████░░░░░░] 75%</code>
    * **AI Summary:** Почему подходит.
    * **Кнопки:** [Подробное описание], [Сгенерировать сопроводительное], [Перейти к вакансии 🔗], [Следующая ➡️].

### 4. Схема Базы Данных (Дополнения)

| Таблица | Описание |
| :--- | :--- |
| `vacancies` | Все вакансии из парсера (глобально). |
| `user_vacancies` | Связующая таблица: `user_id`, `vacancy_id`, `match_score`, `ai_summary`, `cover_letter`, `is_seen`, `is_favorite`. |
| `user_settings` | Твоя текущая таблица (стек, грейд, локация). |

---

### 5. Логика работы (Workflow)

1.  **Парсинг:** Парсер кладет 100 новых вакансий в `vacancies`.
2.  **Скоринг:** Сервис анализатора берет пользователя Vladimir, видит его навыки (Python, PostgreSQL) и прогоняет через них новые вакансии.
3.  **Запись:** Для подходящих вакансий (score > 40%) создаются записи в `user_vacancies`.
4.  **Уведомление:** Бот пишет: "Владимир, нашлось 5 новых вакансий с высоким Match Score!".
5.  **Просмотр:** Владимир открывает раздел **Vacancies** и листает только свои персональные рекомендации.

---

### 6. Технологический стек (рекомендация)
* **Parser:** `BeautifulSoup4` / `Playwright`.
* **AI:** `OpenAI API` (GPT-4o / GPT-4o-mini) или `LangChain`.
* **Database:** `PostgreSQL` (уже используем).
* **Matching:** Можно начать с простого сопоставления слов (TF-IDF / Cosine Similarity), а позже перейти на векторные эмбеддинги (Vector DB).
